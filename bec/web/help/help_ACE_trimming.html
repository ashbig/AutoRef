ACE trimming for the scored sequence

When executed:

ACE trims scored sequence for quality:

  • 1. Quality check for end reads (low quality always rejected by application);
  • 2. Sequence assembly: when quality trimming is requested (user request, rerunable);
  • 3. Trace files quality report ( no action taken);
  • 4. Gap mapper & Low confidence finder Algorithms applied for trimming:

 

End reads quality check:

  • high quality read length, as defined by phred alt trimming (see Phred documentation), should be over 120 bases;
  • it should be over 100 bases of good quality (phred score >= 20) in the first 300 bases of read.

All parameters for this algorithms are hardcoded in ACE.

Sequence assembly:

  • trace is first checked to pass quality criteria (see algorithm for trace files quality report);
  • if trace pass quality check, it is trimmed by applying Phred alt trim algorithm.
Trace files quality report:
  • trace should have at least N bases long;
    • average score for all bases between
  • base X and base Y should be over than score Z, Ambiquous bases are excluded.

Parameters N, X, Y, and Z are defined on application level in ApplicationSettings.property file.
