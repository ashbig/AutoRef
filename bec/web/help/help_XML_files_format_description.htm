XML format for plate upload

Reference sequence file

Reference sequence file describs target sequences for clone(s). All steps on the 'Cloning project settings' menu (see ACE tutorial) MUST be performed before XML file can be written, because some items, like species, should be defined and get their ACE unique IDs before they can be referenced in XML.

Elements

Refsequence-feature

Element refsequence-feature
Description represent one annotation to reference sequence
Attributes Name Type

Optional/

Required

 Default Value Description
name_type String Required n/a Annotation type defined under 'Cloning Project Settings ' menu
name_value String Required n/a Annotation value, e.g., GI for target sequence
url String Optional n/a Link to annotation record, e.g., link to PubMed record for GI
description String Optional n/a Annotation description
Contents none

 

Plate mapping file

XML file describs plate mapping information and cloning strategy. All steps on the 'Cloning project settings' menu (see ACE tutorial) MUST be performed before XML file can be written, because some items, like cloning strategy, should be defined and get their ACE unique IDs before they will be referenced in XML. Example of Plate mapping XML file.

Elements

Clone collection

Element clone-collection
Description Definition of sequencing plate
Attributes Name Type Optional/ Required Default Value Description
userid int Required n/a User unique ID, user must be register with ACE
name String Required n/a Unique plate name
type String Optinal n/a Container type description
projectid int Required n/a Project unique ID, see 'System Settings ' for details
Contents sample, construct

Construct

Element construct
Description container for all isolate originated from the same clone
Attributes Name Type Optional/ Required

 Default Value Description
format enum Required 1 Defins whether stop codon was replace by non-stop codon or not
cloningstrategy_name int Required n/a Cloning strategy ID, see 'System Settings ' for details
refsequenceid int Required n/a Target sequence ID as submited from reference
constructid int Required n/a Construct unique identifier
Contents sample*

Construct format:

  • 0 - stop codon was replaced by non-stop codon;
  • 1 - stop codon preserved;

Sample

Element sample
Description represent one phisical sample on sequencing plate, clone ID is substance identifier.
Attributes Name Type

Optional/

Required

 Default Value Description
sampleid int Required n/a Unique sample ID; can be equal to clone ID depending on customer LIMS.
cloneid int Required n/a Unique clone ID
well String Required n/a Well name in 'A01' annotation
sampletype enum** Required n/a Sample type: see below for possible values
Contents empty
Sample Type:
  • CONTROL_POSITIVE
  • CONTROL_NEGATIVE
  • ISOLATE
  • EMPTY

    • - empty well
Note: All attributes of any type (int, enum,..) MUST be placed in quots in XML file, e.g., <construct constructid='27433' format= '0' cloningstrategyid='5' ....
